## 已知决策

### 1. 选择 FastAPI 作为 Web 框架
- **日期**: 项目初期
- **状态**: 已实施
- **理由**: 
  - 异步支持好，适合 AI 服务的 I/O 密集型场景
  - 自动生成 OpenAPI 文档
  - 类型提示友好

### 2. 采用多租户架构
- **日期**: 项目初期
- **状态**: 已实施
- **方案**: 使用 `tenant_uuid` 前缀隔离会话和资源
- **理由**:
  - 轻量级隔离，无需独立部署
  - 支持配置、技能、RAG 引擎的租户级定制

### 3. 技能系统设计为插件式
- **日期**: 项目初期
- **状态**: 已实施
- **方案**: 内置技能 + 用户自定义技能双目录结构
- **理由**:
  - 保持核心稳定性的同时支持扩展
  - 用户可运行时添加技能
